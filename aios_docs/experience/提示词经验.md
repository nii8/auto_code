# 提示词经验

## 经验 1：通用启动提示词

经验等级：A

适用场景：AIOS 新会话启动。

不适用场景：用户已经在某个任务执行中，只需要继续当前任务。

问题表现：启动时 AI 不知道该读什么文件、按什么流程走。

原因分析：缺少统一入口。

解决方案：使用固定启动提示词。

以后规则：新会话启动优先使用下面提示词。

置信度：高

来源项目：AIOS 初始设计。

日期：2026-06-05

是否可能过期：是

过期检查方式：如果总控入口路径变化，需要更新。

提示词：

```text
请读取 aios_docs/AI项目操作系统总控入口.md，按其中流程启动 AIOS。
```

## 经验 2：目标确认提示词要先给候选，再问用户

经验等级：A

适用场景：目标讨论阶段。

不适用场景：用户已经明确写好目标并要求直接冻结。

问题表现：AI 一口气问太多开放问题，用户负担大。

原因分析：没有先帮用户从材料里提炼候选目标。

解决方案：先给 2-3 个候选目标，再问用户确认哪个是当前阶段目标。

以后规则：目标讨论不直接让用户从零描述，而是“候选目标 + 待确认问题”。

置信度：高

来源项目：AIOS 初始设计。

日期：2026-06-05

是否可能过期：否

过期检查方式：长期有效。
