# AIOS 经验库

> 这个目录用于沉淀跨项目可复用经验。经验只能当参考，不能当裁判；只能帮助提问，不能替代当前项目确认。

## 一、经验库的作用

经验库用于：

```text
减少重复错误
改进初始化流程
改进任务拆解
改进验收标准
改进自动检查
改进 LLM / Codex 调用方式
帮助复盘失败原因
```

经验库不能用于：

```text
替用户决定当前项目目标
把旧项目规则直接套到新项目
跳过当前项目的目标确认
跳过当前项目的规格冻结
```

## 二、经验什么时候有用

经验有用的场景：

```text
遇到相似问题
设计通用流程
生成检查清单
减少用户重复表达
复盘失败原因
提醒历史踩坑
```

典型例子：

```text
第一轮不要改代码
目标先草案再冻结
确定性检查用程序
判断性检查用 LLM
高风险操作要人工确认
```

## 三、经验什么时候有反作用

经验会变成反作用的场景：

```text
新项目和旧项目不同
经验替代用户确认
经验太多导致上下文污染
经验没有适用条件
经验只是情绪，不是规则
经验已经过期
```

典型错误：

```text
以前短视频项目要观点闭环，所以新项目也必须观点闭环。
```

正确做法：

```text
经验库里有“短视频要观点闭环”的经验。
请确认当前项目是否也是短视频类项目，是否适用这条经验？
```

## 四、经验分级

### A 类：强通用经验

几乎所有项目适用。项目启动时应读取 A 类经验，并形成“建议加载清单”。

例如：

```text
目标先确认再冻结
第一轮不要改代码
删除文件前确认
API Key 不写死
确定性检查用程序
```

默认进入启动候选清单，但仍必须在项目初始化阶段展示给用户，由用户选择是否加载。

### B 类：领域经验

只适用于某类项目。

例如：

```text
短视频要观点闭环
SRT 转 MD 不能漏关键内容
播放器要关注交互路径
打卡系统要关注用户留存
```

启动时不直接加载。只有根据当前项目材料判断可能相关，或用户明确要求时，才读取并作为候选启用。必须确认当前项目属于该领域，才能启用。

### C 类：项目特定经验

只适用于某个项目或某个源码。

例如：

```text
某个文件不能删
某个脚本有 bug
某个模型效果不好
某个路径是历史输出
```

启动时不直接加载。只有当源码目录、项目名称、文件路径、业务上下文高度一致，或用户明确要求时，才读取并作为候选参考。默认不启用。

## 五、经验使用流程

AIOS 启动新项目时，必须按这个流程使用经验：

```text
1. 先读取 A 类强通用经验
2. 输出 A 类经验加载候选清单，让用户选择是否加载
3. 再读当前项目材料
4. 判断是否需要读取 B 类领域经验
5. 判断是否需要读取 C 类项目特定经验
6. 对所有候选经验判断适用场景和不适用场景
7. 列出建议启用 / 不建议启用 / 需要用户确认的经验
8. 用户确认后，才写入当前项目规则
```

禁止：

```text
启动时直接加载 B / C 类经验
↓
把旧项目经验当当前项目规则
↓
开始执行
```

## 六、经验条目标准格式

每条经验都必须包含：

```text
## 经验标题

经验等级：A / B / C
适用场景：
不适用场景：
问题表现：
原因分析：
解决方案：
以后规则：
置信度：高 / 中 / 低
来源项目：
日期：
是否可能过期：是 / 否
过期检查方式：
```

尤其重要：

```text
适用场景
不适用场景
经验等级
是否可能过期
```

没有这些字段的经验，不能直接启用。
