# 经验适用性判断规则

> 经验进入新项目前，必须经过适用性判断。

## 一、加载顺序

```text
1. 项目启动时先读取 A 类强通用经验。
2. 输出 A 类经验候选加载清单，让用户选择是否加载。
3. 读取当前项目材料和源码结构。
4. 根据当前项目类型，判断是否需要读取 B 类领域经验。
5. 根据项目路径、名称、文件结构，判断是否需要读取 C 类项目特定经验。
6. 对所有候选经验做适用性判断。
7. 用户确认后，才写入当前项目规则。
```

## 二、判断顺序

```text
1. 当前项目是什么类型？
2. 经验来自什么类型项目？
3. 两者是否同领域？
4. 经验是 A / B / C 哪一类？
5. 经验是否有明确适用场景？
6. 当前项目是否满足适用场景？
7. 当前项目是否命中不适用场景？
8. 经验是否可能过期？
9. 是否需要用户确认？
```

## 三、启用规则

### A 类经验

项目启动时读取，并输出候选加载清单。

即使是 A 类，也要让用户看到“本项目准备加载哪些强通用经验”。如果用户反对，则不启用。

### B 类经验

启动时不直接读取。只有当前项目材料显示可能属于对应领域，或用户明确要求时，才读取。读取后必须确认当前项目属于对应领域。

例如：

```text
短视频经验只能用于短视频 / 视频剪辑类项目。
SRT 转 MD 经验只能用于字幕整理 / 伴随稿类项目。
```

### C 类经验

启动时不直接读取，默认不启用。

只有当源码目录、项目名称、文件路径、业务上下文高度一致，或用户明确要求时，才可读取并建议启用。

## 四、经验输出格式

AIOS 启动时应输出：

```text
A 类启动候选经验：
- E-001：目标先确认再冻结（A 类，建议加载，等待用户确认）

按需读取的领域经验：
- E-002：短视频要观点闭环（B 类，当前项目若是短视频类再读取并确认）

不建议启用经验：
- E-010：某项目路径不能删除（C 类，当前项目路径不同）

需要用户确认：
1. 当前项目是否属于短视频类？
2. 是否启用短视频观点闭环经验？
```

## 五、反作用预警

如果出现下面情况，必须提醒用户：

```text
经验太多，可能污染当前项目。
经验与当前材料冲突。
经验来自旧项目且可能过期。
经验没有适用场景。
经验是情绪描述，不是可执行规则。
```
